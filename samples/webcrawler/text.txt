<div class="post-997 post type-post status-publish format-standard hentry category-cobol category-cobolscript category-javascript category-nodejs category-open-source-projects" id="post-997">
	 <h3 class="storytitle">
			<a href="http://ajlopez.wordpress.com/2013/01/19/cobolscript-4-web-pages-with-templates/" title="Permalink to CobolScript (4) Web Pages with&nbsp;Templates" rel="bookmark">CobolScript (4) Web Pages with&nbsp;Templates</a>
		 </h3>
	<div class="meta">Filed under:  <a href="http://ajlopez.wordpress.com/category/cobol/" title="View all posts in COBOL" rel="category tag">COBOL</a>, <a href="http://ajlopez.wordpress.com/category/cobolscript/" title="View all posts in CobolScript" rel="category tag">CobolScript</a>, <a href="http://ajlopez.wordpress.com/category/javascript/" title="View all posts in JavaScript" rel="category tag">JavaScript</a>, <a href="http://ajlopez.wordpress.com/category/nodejs/" title="View all posts in NodeJs" rel="category tag">NodeJs</a>, <a href="http://ajlopez.wordpress.com/category/open-source-projects/" title="View all posts in Open Source Projects" rel="category tag">Open Source Projects</a> &#8212;  ajlopez @ 3:47 pm </div>

	<div class="storycontent">
		<p><a href="http://ajlopez.wordpress.com/2012/12/29/cobolscript-3-templates/">Previous Post</a></p>
<p>In the previous post I presented CobolScript generating output using templates. It can be used to generate web page output, too. The sample is at:</p>
<p><a title="https://github.com/ajlopez/CobolScript/tree/master/samples/templateweb" href="https://github.com/ajlopez/CobolScript/tree/master/samples/templateweb">https://github.com/ajlopez/CobolScript/tree/master/samples/templateweb</a></p>
<p>The launch program in JavaScript is simple:</p>
</p>
<pre style="background-color:black;color:white;font-size:12pt;">
<span style="color:orange;">var</span> cobs = require<span style="color:cyan;">(</span><span style="color:lightgreen;">'../..'</span><span style="color:cyan;">),</span>
    http = require<span style="color:cyan;">(</span><span style="color:lightgreen;">'http'</span><span style="color:cyan;">),</span>
    fs = require<span style="color:cyan;">(</span><span style="color:lightgreen;">'fs'</span><span style="color:cyan;">);</span>

<span style="color:orange;">var</span> program = cobs.compileTemplateFile<span style="color:cyan;">(</span><span style="color:lightgreen;">'./factorial.cobp'</span><span style="color:cyan;">);</span>

http.createServer<span style="color:cyan;">(</span><span style="color:orange;">function</span><span style="color:cyan;">(</span>req<span style="color:cyan;">,</span> res<span style="color:cyan;">)</span> <span style="color:cyan;">{</span>
    program.run<span style="color:cyan;">(</span>cobs.getRuntime<span style="color:cyan;">({</span> request: req<span style="color:cyan;">,</span> response: res <span style="color:cyan;">}));</span>
<span style="color:cyan;">})</span>.listen<span style="color:cyan;">(</span>8000<span style="color:cyan;">);</span>

console.log<span style="color:cyan;">(</span><span style="color:lightgreen;">'Server started, listening at port 8000'</span><span style="color:cyan;">);</span></pre>
</p>
<p>The key part is the call to compile the file template. It produce a compiled JavaScript function to be invoked. The call of program.run executes the template, with a giving runtime context. The runtime context is build giving the request and response objects of the current incoming request. That runtime derives all the output of the CobolScript program to the response output. So, the template doesn’t know about web request and response. You can see the runtime object as a service provider to the compiled CobolScript program. Its properties can be accessed if you define a LINKAGE SECTION as in classic COBOL. But this feature was not used in this simple sample.</p>
<p>The template file</p>

<pre style="background-color:black;color:white;font-size:12pt;"><span style="color:cyan;">&lt;</span>h1<span style="color:cyan;">&gt;</span>Factorial<span style="color:cyan;">&lt;</span>/h1<span style="color:cyan;">&gt;</span>
<span style="color:cyan;">&lt;</span>p<span style="color:cyan;">&gt;</span>Page generated <span style="color:orange;">by</span> CobolScript<span style="color:cyan;">,</span> <span style="color:orange;">using</span> templates<span style="color:cyan;">&lt;</span>/p<span style="color:cyan;">&gt;</span>
<span style="color:cyan;">&lt;</span>table<span style="color:cyan;">&gt;</span>
<span style="color:cyan;">&lt;</span>tr<span style="color:cyan;">&gt;&lt;</span>th align=<span style="color:cyan;">'</span>right<span style="color:cyan;">'&gt;</span>n<span style="color:cyan;">&lt;</span>/th<span style="color:cyan;">&gt;&lt;</span>th align=<span style="color:cyan;">'</span>right<span style="color:cyan;">'&gt;</span>n<span style="color:cyan;">!
&lt;</span>/th<span style="color:cyan;">&gt;
&lt;</span>/tr<span style="color:cyan;">&gt;</span>
<span style="color:cyan;">&lt;#</span>
<span style="color:orange;">local</span> n.
<span style="color:orange;">perform</span> show-factorial <span style="color:orange;">using</span> n <span style="color:orange;">varying</span> n <span style="color:orange;">from</span> 1 <span style="color:orange;">to</span> 10.
<span style="color:cyan;">#&gt;</span>
<span style="color:cyan;">&lt;</span>/table<span style="color:cyan;">&gt;</span>
<span style="color:cyan;">&lt;#</span>

stop run.

show-factorial <span style="color:orange;">section</span> <span style="color:orange;">using</span> n.
<span style="color:orange;">local</span> result.
<span style="color:orange;">perform</span> factorial <span style="color:orange;">using</span> n <span style="color:orange;">giving</span> result.
<span style="color:cyan;">#&gt;</span>
<span style="color:cyan;">&lt;</span>tr<span style="color:cyan;">&gt;
&lt;</span>td align=<span style="color:cyan;">'</span>right<span style="color:cyan;">'&gt;${</span>n<span style="color:cyan;">}&lt;</span>/td<span style="color:cyan;">&gt;&lt;</span>td align=<span style="color:cyan;">'</span>right<span style="color:cyan;">'&gt;${</span>result<span style="color:cyan;">}

<span style="color:orange;">if</span> n = 1 <span style="color:orange;">then</span> <span style="color:orange;">return</span> n.
<span style="color:orange;">subtract</span> 1 <span style="color:orange;">from</span> n <span style="color:orange;">giving</span> m.

multiply n <span style="color:orange;">by</span> m.

</p>

<h2>January 16, 2013</h2>
<div class="post-993 post type-post status-publish format-standard hentry category-javascript category-nodejs category-open-source-projects" id="post-993">
	 <h3 class="storytitle">
			<a href="http://ajlopez.wordpress.com/2013/01/16/some-projects-with-in-memory-model/" title="Permalink to Some projects with in-memory&nbsp;model" rel="bookmark">Some projects with in-memory&nbsp;model</a>
		 </h3>
	<div class="meta">Filed under:  <a href="http://ajlopez.wordpress.com/category/javascript/" title="View all posts in JavaScript" rel="category tag">JavaScript</a>, <a href="http://ajlopez.wordpress.com/category/nodejs/" title="View all posts in NodeJs" rel="category tag">NodeJs</a>, <a href="http://ajlopez.wordpress.com/category/open-source-projects/" title="View all posts in Open Source Projects" rel="category tag">Open Source Projects</a> &#8212;  ajlopez @ 5:05 pm </div>

	<div class="storycontent">
		<p>I’m a big proponent of having models (domain models) residing in memory. Of course, in the appropriate context. There are many problems that could have a better solution if we forgot the “use the database, Luke” approach. One notable solution/project is <a href="http://redis.io/">Redis</a>, that using a single thread implements a complete and powerful key-value store in memory, with options for persistence and clustering (redis cluster still under development).</p>
<p>I’m playing implementing personal projects using an in-memory model. There are:</p>
<p><a href="https://github.com/ajlopez/SimpleMemolap">SimpleMemolap</a>: Multidimensional OLAP-like model, in JavaScript/Node.js with a web sample.</p>
<p><a href="https://github.com/ajlopez/SimpleMemolap">Memolap</a>: Multidimensional OLAP-like model, in C#. Web sample in progress.</p>
<p><a href="https://github.com/ajlopez/SimplePermissions">SimplePermissions</a>: Subjects, Roles, Permissions, Contexts, all in memory.</p>
<p><a href="https://github.com/ajlopez/AjKeyvs">AjKeyvs</a>: Redis-like in-memory key-value store, in C#. I should add the client protocol. I should review the internal algorithms, but all was coded using TDD, ready for refactors.</p>
<p><a href="https://github.com/ajlopez/SimpleStore">SimpleStore</a>: Simple Key Value Store in memory, a la Redis, JavaScript/Node.js. In process or exposed to remote clients using additional modules. Work in Progress.</p>
<p><a href="https://github.com/ajlopez/SimpleQueue">SimpleQueue</a>: in-memory queue implementation, JavaScript/Node.js. It could be used in process or acceded via exposing the methods to remote calls.</p>
<p><a href="https://github.com/ajlopez/SimpleTags">SimpleTags</a>: having items with arbitrary data, tagged with strings or key-value objects.</p>
<p>Maybe I could add <a href="https://github.com/ajlopez/SimpleRules">SimpleRules</a>: rule engine where the facts reside in-memory (many rule engines use directly the memory to represent the world state).</p>
<p>And even I could mention: <a href="https://github.com/ajlopez/SimpleBoggle">SimpleBoggle</a>, where the list of words to resolve a <a href="http://en.wikipedia.org/wiki/Boggle">Boggle game</a> it’s loaded in memory <img src='http://s1.wp.com/wp-includes/images/smilies/icon_wink.gif?m=1129645325g' alt=';-)' class='wp-smiley' /> </p>
<p>To be started:</p>
<p><a href="https://github.com/ajlopez/SimpleDatabase">SimpleDatabase</a>: in-memory database, with SQL-like query language, JavaScript/Node.js</p>
<p>Most of these projects could be extended to have pluggable persistence. But now I’m interested in having something that works to play with. I’m having fun implementing internal algorithms, trees, search, inverted indexes, etc. And practicing TDD every day.</p>
<p>Keep tuned!</p>

<h2>January 2, 2013</h2>
<div class="post-984 post type-post status-publish format-standard hentry category-compilers category-javascript category-nodejs category-open-source-projects category-programming-languages category-simplescript" id="post-984">
	 <h3 class="storytitle">
			<a href="http://ajlopez.wordpress.com/2013/01/02/simplescript-1-first-ideas/" title="Permalink to SimpleScript (1) First&nbsp;ideas" rel="bookmark">SimpleScript (1) First&nbsp;ideas</a>
		 </h3>
	<div class="meta">Filed under:  <a href="http://ajlopez.wordpress.com/category/compilers/" title="View all posts in Compilers" rel="category tag">Compilers</a>, <a href="http://ajlopez.wordpress.com/category/javascript/" title="View all posts in JavaScript" rel="category tag">JavaScript</a>, <a href="http://ajlopez.wordpress.com/category/nodejs/" title="View all posts in NodeJs" rel="category tag">NodeJs</a>, <a href="http://ajlopez.wordpress.com/category/open-source-projects/" title="View all posts in Open Source Projects" rel="category tag">Open Source Projects</a>, <a href="http://ajlopez.wordpress.com/category/programming-languages/" title="View all posts in Programming Languages" rel="category tag">Programming Languages</a>, <a href="http://ajlopez.wordpress.com/category/simplescript/" title="View all posts in SimpleScript" rel="category tag">SimpleScript</a> &#8212;  ajlopez @ 2:38 pm </div>

	<div class="storycontent">
		<p>The latest two week, I was busy writing <a href="https://github.com/ajlopez/CobolScript">CobolScript</a>, my COBOL compiler to JavaScript (see <a href="http://ajlopez.wordpress.com/category/cobolscript/">my posts</a>). I have console sample programs, and dynamic pages samples running on Node.js (see <a href="https://github.com/ajlopez/CobolScript#samples">samples</a>). The web samples are using the simple http node.js module, or my <a href="https://github.com/ajlopez/SimpleWeb">new SimpleWeb module</a>, a simple middleware layer a la Connect. I started to write a Python to JavaScript compiler, see <a href="https://github.com/ajlopez/Py2Script">Py2Script</a>, too. But now, after those training projecs (my first ones that are compiling to JavaScript using JavaScript), I want to push the envelope and write a simple script compiler, I named SimpleScript (see <a href="https://github.com/ajlopez/SimpleScript">repo</a>).</p>
<p>The key points:</p>
<p>- It compiles to JavaScript, so, it’s JavaScript-oriented. It’s not a script language to be implemented in different technologies (.NET, Java, JavaScript). It’s totally oriented to JavaScript semantic.</p>
<p>- I love C programming language tradition, but this time, I want no semicolons and curly braces. I want a syntax more Python/Ruby oriented.</p>
<p>- No indentation “hell”. Ok, I like Python, but indentation as part of the syntax, it’s not my preferred way.</p>
<p>- No command separator (no semicolon or something else) except new line or syntax. That is, I could write</p>
<p><font face="Consolas"><strong>if</strong> a &lt; 1 a = 1</font></p>
<p>or</p>
<p><font face="Consolas"><strong>if</strong> a &lt; 1      <br />&#160;&#160; a = 1      <br /><strong>end</strong></font></p>
<p>Notice the use of <strong>end</strong>.</p>
<p>You CANNOT write (semicolon is not a separator):</p>
<p><font face="Consolas"><strong>if</strong> a &lt; 1 a=1; b=2</font></p>
<p>You MUST write:</p>
<p><font face="Consolas"><strong>if</strong> a &lt; 1      <br />&#160; a = 1      <br />&#160; b = 2      <br /><strong>end</strong></font></p>
<p>- No parenthesis around conditions (see the above if example).</p>
<p>- Only<strong> for … in</strong> , to be discussed. I want to have for-in as in JavaScript, but with some variant to access directly the values instead the names/indexes of an object or array. Something like</p>
<p><strong>for</strong> k <strong>in</strong> myarray</p>
<p>iterates over myarray indexes.</p>
<p><strong>for</strong> k <strong>in</strong> myarray <strong>values</strong></p>
<p>iterates over myarray values, directly. Range expression will be supported:</p>
<p><font face="Consolas"><strong>for</strong> k <strong>in</strong> 0..n</font></p>
<p>- Loops with <strong>continue</strong>, <strong>break</strong>. The main loop construction is the <strong>while</strong>.</p>
<p>- Functions as first class citizens.</p>
<p>- <strong>function</strong> keyword will be used to define anonymous functions. Maybe (to be discussed) I will use <strong>define</strong> keyword to define named functions.</p>
<p>- Function invocation with explicit parenthesis (forget Ruby convention, go for something like Python 3.x).</p>
<p>- Array access with <strong>[]</strong> (forget Basic programming convention of using parenthesis).</p>
<p>- External variables. I learnt this from my CobolScript work: the linkage section is useful. An external variable is something injected at runtime (not a global one), and it can be provided in the call to the program. For example, <strong>print</strong> function could be an external one, so the calling program could redirect the print to the console, or to a buffer or to the web response. at will.</p>
<p>- Global variables should be declared explicitly. All non-declared variable is considered local (to the function, unit where it is used).</p>
<p>- Functions has closures as in JavaScript. In contrast, AFAIK, Python use explicit declaration to access outer variables in a function. I like JavaScript automatic access to outer variables, so I will keep it.</p>
<p>- Async constructions. I added this to CobolScript, and I found it useful. It&#8217;s something like the await/async in C# 5.0</p>
<p>- Run on the browser and on Node.js.</p>
<p>- Class support: to be discussed (the only key use case I have, is the game client coding for my game project).</p>
<p>I know, there are other implementations, like CofeeScript. But I want to training me, on JavaScript, Node.js and TDD.</p>
<p>Keep tuned!</p>
